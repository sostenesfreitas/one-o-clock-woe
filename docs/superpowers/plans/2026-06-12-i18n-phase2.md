## Established conventions (carry over from Phase 1 — apply to every task)

- These render functions return **JS template literals** → translate with inline `t('key')` / `t('key',{x})`. Static `<body>` markup (the `<dialog>`s) uses `data-i18n` / `data-i18n-attr`.
- `t()` returns **plain text** — NEVER put HTML (`<b>`,`<br>`,`<span>`) inside a locale value; keep HTML in the template, translate only the text fragment. Drop `<br>` line-breaks inside values.
- Pass user input pre-escaped: `t('key',{q: escapeHtml(...)})`. Keep `title="${t(...)}"` values free of `"`.
- Interpolation params MUST reuse the SAME expressions they replaced. Numbers/math never change.
- **Game jargon stays literal English** (NOT keyed): item names (Card/White/Black/Illusion — incl. `AR_ITEM_META`), Knight/job names, GL/Overrun/WoE/party, color groups.
- **Reuse existing keys** where the value already exists (e.g. `common.confirm`, `common.delete`, `common.status`, `status.short`, `auction.field_main`/`field_sub`, `auction.not_dragged`). Grep `LOCALES` before adding.
- Key naming: `area.item` snake/kebab — `summary.*`, `ai.*`, `leave.*`, `wheel.*`, `users.*`, `ar.*`, `dialog.*`, `toast.*`, `header.*`.
- **No `t` shadow** exists in any Phase-2 function (verified by inventory) — safe to call `t()` directly.
- CLAUDE.md gate every commit: `node test/parse-check.js` → `PARSE OK`; `node test/run.js` → `0 failed`; behavior change ⇒ update tests same commit; never merge to `main` without user confirmation.

## Day-name LOGIC safety (critical, applies to Leave + Auction-Request)

The Thai day names อังคาร/พฤหัส/อาทิตย์ are **display-only**. `isEventDay(iso)` (app.html ~3292) decides events from **numeric** `bkkDow()` (0/2/4), NEVER from a day-name string. So:
- Translate day-name **display** (via the new `dowName()` helper / `LOCALES`).
- Do NOT touch `isEventDay`, `bkkDow`, or any numeric weekday comparison.
- Day names that appear inside `// comments` / docstrings stay as-is (comments aren't translated).

## Per-surface procedure (every Task B–I follows this)

1. Read the target function(s); list every rendered Thai string (skip comments).
2. Add keys to BOTH `LOCALES["pt-BR"]` and `LOCALES["en"]` (reuse where possible); record each in `docs/i18n-glossary.md`.
3. Replace each literal with inline `t()` (template renderers) or `data-i18n` (static `<dialog>` markup).
4. Update any existing test asserting a changed Thai literal → new pt-BR text or stable structural token, SAME commit (numbers preserved).
5. Add one `[i18n]` test locking a few of the surface's keys in both locales.
6. `node test/parse-check.js` → `PARSE OK`; `node test/run.js` → `0 failed`.
7. Commit. Then spec-compliance review → code-quality review → fix loop.

```js
const DOW_LOCALES = {
  "pt-BR": ["Domingo","Segunda","Terça","Quarta","Quinta","Sexta","Sábado"],
  "en":    ["Sunday","Monday","Tuesday","Wednesday","Thursday","Friday","Saturday"],
};
const DOW_SHORT_LOCALES = {
  "pt-BR": ["Dom","Seg","Ter","Qua","Qui","Sex","Sáb"],
  "en":    ["Sun","Mon","Tue","Wed","Thu","Fri","Sat"],
};
const MONTH_LOCALES = {
  "pt-BR": ["jan.","fev.","mar.","abr.","mai.","jun.","jul.","ago.","set.","out.","nov.","dez."],
  "en":    ["Jan","Feb","Mar","Apr","May","Jun","Jul","Aug","Sep","Oct","Nov","Dec"],
};
function dowName(dow)      { return (DOW_LOCALES[currentLocale]||DOW_LOCALES["pt-BR"])[dow] ?? ""; }
function dowShort(dow)     { return (DOW_SHORT_LOCALES[currentLocale]||DOW_SHORT_LOCALES["pt-BR"])[dow] ?? ""; }
/* Gregorian, BKK, locale month — replaces fmtThaiDate's Buddhist-era output. */
function fmtDate(iso) {
  const [y,m,d] = String(iso).split("-").map(Number);
  if (!y||!m||!d) return String(iso);
  const mon = (MONTH_LOCALES[currentLocale]||MONTH_LOCALES["pt-BR"])[m-1];
  return d + " " + mon + " " + y;
}
```

If grep shows `fmtThaiDate`/`DOW_TH` are ONLY used in display sites you will convert, you may remove them at the end of Task C/F — report which.)
